# ფიქსი: მომხმარებელი როლის მინიჭება

## პრობლემა
- **GetAllAdmins** - ახლად დამატებული ადმინი არ მოდიოდა
- **GetAllMembers** - რეგისტრირებული მომხმარებლები არ მოდიოდნენ

## მიზეზი
მომხმარებელი რეგისტრაციისა და დამატების დროს მათ **წვდომის როლი არ მიენიჭებოდათ**. ვინაიდან repository მეთოდი `GetUsersByRoleAsync()` ფილტრავს მომხმარებლებს მათი როლის მიხედვით, დაუნიჭებელი მომხმარებლები არ ჩნდებიან.

## გადაწყვეტა

### 1. IRoleRepository გაფართოვება
დამატებული მეთოდი:
```csharp
Task<Role?> GetByRoleNameAsync(string roleName);
```

### 2. UserService შეცვლა
რეგისტრაციის მეთოდში ახლა დავამატეთ კოდი:
- ახალი მომხმარებელი ავტომატიკად მიენიჭება **"Customer"** როლი

### 3. AdminService შეცვლა
`AddUser` მეთოდში ახლა დავამატეთ კოდი:
- ადმინის მიერ დამატებული მომხმარებელი ავტომატიკად მიენიჭება **"Customer"** როლი

### 4. ახალი მიგრაცია
ფაილი: `20260331000001_ensure_roles_exist.cs`
- ახლის რეგისტრაციის დროს შექმნილი ახალი ადმინი ანიჭება:
  - **Admin** როლი (თუ ადმინი არის)
  - **Customer** როლი (ჩვეულებრივ მომხმარებელზე)
  - **Trainer** როლი (დამწვრილებით)
